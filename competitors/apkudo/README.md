# Apkudo

**Company:** Apkudo (Baltimore, MD)
**Products:** RSA (Robotic Standalone Automation) + Full Automation Line
**Status:** Deployed globally

## Hardware

Two product tiers:

### RSA (Standalone Unit)

| Spec | Value |
|------|-------|
| Form factor | Compact, fits through standard doorways (~36" wide) |
| Throughput | 30+ devices/hr |
| Operators | 2 (one ingress, one egress) |
| Functions | Ingress → functional → cosmetic → data erase → egress |

### Full Automation Line

| Spec | Value |
|------|-------|
| Form factor | Modular processing cells linked by conveyors |
| Stages | 8+ (receiving, cleaning, cosmetic, display, touch, camera/hw, audio, data clearing, protection) |
| Throughput | 250-300 devices/hr |
| Operators | Minimal (ingress/egress only) |
| Tests per device | 50+ distinct tests |
| Deployed | Dallas facility (270/hr), Cyculus Services Germany (~300/hr, 100K/month, 23hr/day) |

## Why This Doesn't Work For Us

- Full line requires warehouse space and conveyor infrastructure
- 8+ modular cells = massive investment
- Designed for enterprise-scale operations (100K+/month)
- Even the RSA requires 2 operators

## What We Can Learn

- Modular cell approach allows scaling incrementally (add cells as volume grows)
- 50+ tests per device shows the value of combining cosmetic + functional in one pass
- 300/hr at Cyculus proves high-throughput automated grading works in production
- "Fits through standard doorways" is a smart form factor constraint — keep portability in mind
